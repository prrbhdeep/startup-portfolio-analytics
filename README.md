# Startup Portfolio Analytics System

End-to-end data warehouse + ML risk assessment system for a simulated incubator
portfolio of **110 startups** (24 months of KPIs each). Built in three phases,
each in its own folder.

## 📁 Project Structure

```
├── data/                    Shared datasets (generated by Phase 1)
├── phase1_data_warehouse/   Phase 1 — SQL warehouse & data generation
├── phase2_risk_model/       Phase 2 — ML risk assessment
└── phase3_dashboards/       Phase 3 — Interactive dashboards (in progress)
```

## Phase 1 — Data Warehouse ✅
**Goal:** Centralized SQL + Python warehouse consolidating growth, financial, and PMF metrics.
- `generate_data.py` — synthetic generator: 110 startups × 24 months = 2,640 records, with realistic mess (4% missing values, founder reporting noise, overlapping health profiles)
- `schema.sql` — warehouse schema with derived views (`latest_health`, `burn_efficiency`)
- Output: `data/portfolio.db` (SQLite), CSVs

**Run:** `cd phase1_data_warehouse && python generate_data.py`

## Phase 2 — Risk Assessment Model ✅
**Goal:** KPI-driven multi-model system flagging the bottom 10% at-risk startups monthly.
- Feature engineering from trailing 6 months (growth, churn, runway, burn efficiency, volatility)
- **Logistic regression** (supervised) + **Isolation Forest** (anomaly detection), combined risk score
- Validated against held-out ground-truth labels

**Results:** ROC-AUC 0.957 · Precision@10% = 0.73 · Recall of at-risk cohort = 0.53

**Run:** `cd phase2_risk_model && python risk_model.py`

## Phase 3 — Dashboards 🚧
**Goal:** Interactive dashboards tracking ROI, scalability, and sector trends.
- Plotly dashboard (Python) → Tableau Public recreation

## Key Design Decision
The first data version produced a perfect AUC of 1.0 — a red flag that the synthetic
classes were too separable. I added reporting noise, missing values, and archetype
overlap to mirror real self-reported startup data, then re-evaluated. Realistic
data → realistic (and defensible) metrics.

## Stack
Python (pandas, scikit-learn) · SQL (SQLite) · Plotly · Tableau
